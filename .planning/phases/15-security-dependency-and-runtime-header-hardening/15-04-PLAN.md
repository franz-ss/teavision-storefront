    1. In `src/lib/env/server.ts`, add named exports:
       - `getRateLimitTrustedIpHeader()` returning `x-forwarded-for`, `x-real-ip`, `cf-connecting-ip`, or `undefined`. It reads `RATE_LIMIT_TRUSTED_IP_HEADER`, trims/lowercases it, and accepts only those three values.
       - `isRateLimitProductionExplicit()` returning true when production is false, or when production is true and either `RATE_LIMIT_EXTERNAL_PROTECTION=true` or both `RATE_LIMIT_ALLOW_MEMORY_FALLBACK=true` and `getRateLimitTrustedIpHeader()` returns a configured header.     2. In `src/lib/rate-limit/index.ts`, update `getClientIpFromHeaders(headers)`:
       - If `getRateLimitTrustedIpHeader()` returns `x-forwarded-for`, use only the first comma-separated `x-forwarded-for` value.
       - If it returns `x-real-ip`, use only `x-real-ip`.
       - If it returns `cf-connecting-ip`, use only `cf-connecting-ip`.
       - If no trusted header is configured, preserve current local/test fallback precedence.

    3. In `checkRateLimit()`, fail closed in production when `isRateLimitProductionExplicit()` is false. Return a limited result with `limited: true`, `remaining: 0`, and `resetAt: Date.now() + safeWindowMs`. Do not throw. This lets public forms/search return existing user-safe rate-limit copy.

    4. Add `src/lib/rate-limit/index.test.ts` covering:
       - production with no `RATE_LIMIT_EXTERNAL_PROTECTION` and no `RATE_LIMIT_ALLOW_MEMORY_FALLBACK` returns `limited: true`
       - production with `RATE_LIMIT_EXTERNAL_PROTECTION=true` allows normal counting
       - production with `RATE_LIMIT_ALLOW_MEMORY_FALLBACK=true` and `RATE_LIMIT_TRUSTED_IP_HEADER=x-forwarded-for` allows normal counting
       - production with `RATE_LIMIT_ALLOW_MEMORY_FALLBACK=true` and no trusted header still returns `limited: true`
       - `RATE_LIMIT_TRUSTED_IP_HEADER=cf-connecting-ip` ignores spoofed `x-forwarded-for`
       - no trusted header preserves existing local/test precedence

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public browser to form/search actions | Untrusted clients can spam form Server Actions and search suggestions |
| Proxy/CDN to app headers | App must trust only the deployment-approved client IP header |
| Provider errors to logs | External provider failures must not leak submitted PII or payloads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-07 | Denial of Service | public form/search surfaces | mitigate | Fail closed in production unless provider protection or intentional fallback is explicit |
| T-15-08 | Spoofing | client IP detection | mitigate | Add tested trusted-header selection and documentation |
| T-15-09 | Information Disclosure | provider error logging | mitigate | Replace raw provider/error logs in touched paths with structured redacted events |

</threat_model>
