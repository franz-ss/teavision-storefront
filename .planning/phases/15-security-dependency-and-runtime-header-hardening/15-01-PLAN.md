## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry to local dependency graph | Package upgrades and overrides alter executable code used by runtime and tooling |
| dev tooling to production runtime | Some audit paths are tooling-only, but direct runtime dependencies such as `next` affect production |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Elevation of Privilege / Denial of Service | vulnerable `next` runtime package | mitigate | Upgrade Next and eslint config to patched Next 16 line, then verify build and integration tests |
| T-15-02 | Supply Chain | transitive dev tooling advisories | mitigate | Use targeted upgrades/overrides, audit after install, and document any residual dev-only moderate findings |
| T-15-SC | Supply Chain | lockfile mutation | mitigate | Update lockfile only through pnpm commands and verify `pnpm audit`, lint, typecheck, build, and tests |

</threat_model>
