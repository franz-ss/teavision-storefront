src/lib/sanity/queries/blog.ts — defaultBlogListingQuery, two identical occurrences of the
unsafe exclusion subquery (line 175 in the "articles" filter, line 184 in the "totalCount" filter):

  !(_id in *[_type == "blog" && slug.current == $blogHandle][0].featuredPosts[]._ref)

The ref list `*[_type == "blog" && slug.current == $blogHandle][0].featuredPosts[]._ref` evaluates
to null (not []) when the blog document has no featuredPosts attribute. Under GROQ three-valued
logic, `_id in null` is null and `!null` is null, so the predicate fails for every article.

src/components/blog/hero/hero.tsx:42 — current unsafe guard:

  const hasLqip = heroImage?.lqip != null

This is true for lqip: '' (empty string), which then drives:
  placeholder={hasLqip ? 'blur' : 'empty'}
  blurDataURL={hasLqip ? (heroImage?.lqip ?? undefined) : undefined}
producing placeholder="blur" with blurDataURL="" — next/image throws at render for remote images.

src/components/ui/article-card/article-card.tsx:100 — the correct, already-safe reference pattern
to mirror (truthy evaluation of lqip):
  placeholder={article.featuredImage.lqip ? 'blur' : 'empty'}
