<locked_decisions>

- Phase 21 is CMS-only: do not edit `src/app/(storefront)/page.tsx`, homepage rendering components, metadata, JSON-LD, cache behavior, webhook behavior, or Next image behavior.
- The sibling `../teavision-cms` Studio owns homepage schemas and seed tooling.
- Use one fixed `homePage` singleton document with document ID `homePage`; do not add a generic page-builder section array.
- Preserve current homepage section order from `src/app/(storefront)/page.tsx`.
- Do not repeat `homepage` in filenames inside `schemaTypes/objects/homepage/`; use `section.ts`, `link.ts`, `hero.ts`, etc.
- Sanity schema type names are global, so use compact names such as `homeHero`, `homeSection`, and `homeImageCard`.
- The seed command must dry-run by default and require `--execute` before writes.
- Any SEO/PageSpeed regression is forbidden; this phase should have no production SEO/PageSpeed impact because it does not change storefront output.
- Do not remove or reformat the pre-existing untracked `../teavision-cms/pnpm-workspace.yaml`.
</locked_decisions>

<threat_model>

| Threat                                            | Risk                                                               | Mitigation                                                                                                                 |
| ------------------------------------------------- | ------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| T-21-01 Accidental multiple homepage documents    | Editors can create drift between two homepages                     | Add a fixed `homePage` document ID and a singleton Studio structure item; filter `homePage` out of generic document lists. |
| T-21-02 Page-builder scope creep                  | Flexible section arrays create layout and QA risk                  | Use fixed fields in current homepage order and only minimal reusable objects.                                              |
| T-21-03 Seed leaks secrets or unpublished content | Logs could expose write tokens or raw document content             | Reuse env helpers, log only counts/status/document ID, and never log tokens, signatures, raw images, or full seeded JSON.  |
| T-21-04 Partial seed masks missing assets         | Homepage document appears seeded while required images are missing | Required local image paths must fail the seed in execute mode if missing.                                                  |
| T-21-05 Published storefront regression           | CMS prep accidentally changes `/` output                           | Do not edit storefront runtime files in Phase 21; verification is limited to CMS tests/build.                              |

</threat_model>
