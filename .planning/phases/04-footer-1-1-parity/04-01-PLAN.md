<risks_and_edge_cases>

- The live footer includes a `Login` link to `https://mrtea.com.au/account/login`; this may be stale, but must be preserved for 1:1 parity unless the user explicitly changes scope.
- The live footer includes hidden SEO keyword links. The headless implementation intentionally removes them after review; this deviation must stay documented.
- The live footer uses Roboto and raw legacy colors. The Next implementation must express equivalent values through design tokens/utilities without raw hex class names.
- Some live hrefs include trailing slashes and typo-like labels such as `import herbs to australias`; do not correct them during this phase.
- Payment SVGs are long and easy to damage. Keep them isolated if needed and verify accessible names.
- Newsletter backend differs from Shopify's native customer form. Preserve current server action safety while matching visible UI.
- Existing uncommitted work is present in unrelated files; avoid broad formatting or refactors.

</risks_and_edge_cases>
