<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Preview URL -> Draft Mode cookie | A URL containing a shared secret can enable a browser cookie that changes homepage rendering. |
| Draft Sanity API -> storefront render | Unpublished CMS content crosses from Sanity into a server-rendered preview response. |
| Published cache -> draft render | Published cached data and draft data coexist in the same route and must not contaminate each other. |
| Sanity webhook -> Next revalidation | External POST payloads can expire cache tags after signature verification. |
| Release evidence -> rollout decision | Human-supplied PSI and smoke-test proof controls whether the release proceeds. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-23-01 | Spoofing | `src/app/api/draft/route.ts` | mitigate | Require `SANITY_PREVIEW_SECRET` with at least 32 random characters, return 500 when server configuration is missing, compare caller and expected secrets with `crypto.timingSafeEqual()` plus a same-length Buffer guard, document production rate limiting/platform protection for failed attempts, enforce exact homepage slug allowlist, and confirm document existence before `draftMode().enable()`. |
| T-23-02 | Tampering | Preview slug redirect | mitigate | Accept only exact slug `/`; reject absolute, protocol-relative, encoded-host, missing, or unsupported slugs before redirecting. |
| T-23-03 | Information Disclosure | Homepage metadata, JSON-LD, robots, sitemap | mitigate | Keep `generateMetadata()`, JSON-LD, sitemap, robots, and cache-tagged published helper on the published data path only; add tests proving draft body does not alter SEO output. |
| T-23-04 | Information Disclosure | Sanity draft content | mitigate | Keep draft helper token-backed, uncached, `stega: false`, and fail-loud. Never log draft field values and never fall back to published after draft validation failure. |
| T-23-05 | Tampering | Published homepage cache | mitigate | Keep `getHomepage()` as the only cached/tagged published helper; ensure `getDraftHomepage()` does not call `cacheTag()` or `cacheLife()`. |
