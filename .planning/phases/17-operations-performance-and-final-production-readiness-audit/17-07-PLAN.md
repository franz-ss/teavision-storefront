<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disabled SEO evidence to enabled SEO evidence | The audit must not reuse a noindex build to prove launch indexing |
| Build-time env to metadata output | Static metadata can bake `DISABLE_INDEXING`, so launch mode must be represented before `next build` |
| Search Console proof to automated score | Search Console remains owner-gated and cannot be faked by local route probes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-16 | Repudiation | final readiness SEO evidence | mitigate | Distinct lifecycle profiles record whether disabled or enabled SEO was probed |
| T-17-17 | Spoofing / Repudiation | owner-gated Search Console proof | mitigate | Keep Search Console as `pending`, `owner-blocked`, or `approved` only with dated owner evidence |
| T-17-18 | Tampering | noindex safety helper | mitigate | Preserve `withNoindexRobots`; switch launch evidence through env profile instead of deleting safety controls |

</threat_model>
