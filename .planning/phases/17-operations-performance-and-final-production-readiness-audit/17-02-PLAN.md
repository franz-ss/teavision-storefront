<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Runtime error to observability provider | Server errors may include sensitive request context and must be redacted |
| Revenue/provider call sites to logs | Checkout, account, provider, webhook, and form boundaries can expose PII or provider payloads |
| Vercel/Sentry operator surfaces to launch decisions | Operators need enough signal to respond without storing raw customer data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-04 | Information Disclosure | logging/redaction helper | mitigate | Unit-test token, PII, checkout URL, body, and payload redaction |
| T-17-05 | Repudiation | route/action failures | mitigate | Next `onRequestError` forwards server request errors to configured monitoring |
| T-17-06 | Information Disclosure | provider and form call sites | mitigate | Replace raw console calls with typed redacted `logEvent` calls |

</threat_model>
