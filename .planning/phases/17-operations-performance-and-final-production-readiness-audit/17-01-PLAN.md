<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public browser to health route | Untrusted users can call health repeatedly and must not receive secrets or diagnostics |
| Operator machine to deep readiness probe | Private checks can inspect config but must not print secret values |
| Automated readiness to owner/admin gates | Code readiness must not pretend owner-controlled Shopify/admin evidence is automated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Information Disclosure | `/api/health` | mitigate | Public response has exact safe fields and forbidden-string tests |
| T-17-02 | Information Disclosure | deep readiness probe | mitigate | Print presence/status only, never secret values or raw provider payloads |
| T-17-03 | Repudiation | owner-gated launch proof | mitigate | Owner gates have explicit approved/pending/owner-blocked states and evidence fields |

</threat_model>
