<threat_model>
- T-14-01 OAuth CSRF/replay, severity high: state, nonce, and PKCE are generated with secure randomness, stored only in the pending-auth HttpOnly cookie, validated before token exchange/use, and cleared on success or failure.
- T-14-02 Token exposure, severity high: access, refresh, and ID tokens remain inside sealed server-owned cookies and are never passed to client components, Storybook args, console logs, or error messages.
- T-14-03 Session fixation/expiry, severity medium: expired or unrefreshable sessions clear local cookies and redirect to `/account/login` with a safe path-only return target and sign-in-again flash message.
- T-14-04 Open redirect, severity high: `returnTo` is accepted only as a same-origin path beginning with `/` and not `//`; unsupported values normalize to `/account`.
- T-14-07 Customer data caching leak, severity high: Customer Account API fetches default to `cache: 'no-store'` and are not wrapped in `use cache`, `cacheTag()`, or product/collection cache helpers.
</threat_model>
