## Scope and completion policy

Apply OWASP ASVS Level 1. Any unresolved High threat blocks completion. This phase changes semantic markup only; it must neither widen the trusted Shopify HTML surface nor replace a browser-native disclosure with custom client interaction.

| ID | Threat | Severity | Mitigation | Verification |
| --- | --- | --- | --- | --- |
| T-25-01 | A new collection variant weakens the trusted Shopify HTML boundary or bypasses branding. | High | Reuse the sole sanitizer's existing allowlists, schemes, link/image transforms, class filtering, and table wrappers. Return branded `SanitizedHtml` only from the sanitizer; do not introduce casts or another HTML sink. | Unit tests retain compact assertions and assert collectionStory output; static review confirms no new `dangerouslySetInnerHTML` or `as SanitizedHtml` in application paths. |
| T-25-02 | A shared compact transform changes product descriptions while fixing collections. | High | Add a collection-only entry point and update only the non-rich-hero collection read-more call. Leave the compact API, transform, import, and product call unchanged. | Compact unit expectation and PDP static render retain imported H1/H2 -> compact H3 output; source inspection and collection route test prove call-site isolation. |
| T-25-03 | ARIA or scripted disclosure changes regress native keyboard, state, or accessibility semantics. | High | Keep native `details` and `summary`; nest an H2 inside summary without role/button, aria-expanded, handlers, or client state. Preserve first-panel `open`, focus classes, and decorative aria-hidden chevron. | PDP markup assertions prove details > summary > h2, exactly one H1, first open, and no forbidden role/ARIA attributes; Storybook browser/manual check exercises keyboard toggle and focus. |
| T-25-04 | Collection heading promotion changes rich-hero or visual hierarchy unexpectedly. | Medium | Parse `richHero` first and invoke the sanitizer only within the `hasRichDescription && !richHero` branch; retain source-aware compact visual classes, and do not modify `parseCollectionRichHero`/`CollectionRichHero`. | Collection static markup asserts H2/H3 after `#product-grid`; a mocked sanitizer proves rich-hero input invokes it zero times; and rich-hero assertions remain green. |

Residual risk: Shopify author input can still contain semantically odd H5/H6 content; that is intentionally unchanged by this narrow phase and remains bounded by the established sanitizer.
</threat_model>
