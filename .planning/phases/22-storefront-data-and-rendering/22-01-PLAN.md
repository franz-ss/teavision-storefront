## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Sanity CMS -> storefront server operation | Published CMS content crosses from an external content system into route rendering and metadata. |
| Sanity image asset metadata -> Next Image | External image URL, dimensions, crop, hotspot, LQIP, and alt text are normalized before rendering. |
| CMS SEO fields -> Next metadata | CMS title, description, canonical, and noIndex influence crawler-visible output. |
| CMS links -> storefront anchors/buttons | Editor-authored hrefs become public navigation affordances. |
| Homepage route -> Shopify/cart/contact actions | CMS rendering must not weaken commerce ownership or form action security boundaries. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-22-01 | Tampering | `src/lib/sanity/home-page.ts` | mitigate | Validate required `homePage` document, fixed canonical `/`, required SEO fields, required counts, and required image URL/dimensions; throw on invalid content. |
| T-22-02 | Information Disclosure | `src/app/(storefront)/page.tsx` metadata | mitigate | Apply CMS noIndex intent through `withNoindexRobots()` so global launch noindex controls still win. |
| T-22-03 | Denial of Service | Sanity image rendering | mitigate | Reject missing image dimensions before render and use bounded `getSanityImageUrl()` widths/quality to avoid oversized images and layout shift. |
| T-22-04 | Spoofing | CMS-authored hrefs | mitigate | Mirror CMS href allowlist: `/`, `https://`, `mailto:`, and `tel:` only; reject invalid links instead of rewriting. |
| T-22-05 | Elevation of Privilege | Homepage rendering components | mitigate | Keep parent sections as Server Components and preserve client code only in existing interactive leaves. |
| T-22-06 | Tampering | Commerce boundaries | mitigate | Sanity query/view model excludes product price, cart, checkout, and discount authority; Shopify operations and existing Server Actions remain authoritative. |
| T-22-07 | Repudiation | Static fallback/content drift | mitigate | Route imports no static homepage content; missing CMS content fails loudly and tests assert no static SEO/content fallback. |
| T-22-08 | Denial of Service | Next Cache Components | mitigate | Use cached homepage operation with `cacheTag()`/`cacheLife()` and no request runtime APIs inside cached scope; build verifies Next 16 rules. |
</threat_model>
