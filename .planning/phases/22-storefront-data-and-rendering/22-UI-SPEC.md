# Phase 22 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — existing manual Tailwind 4 token system |
| Preset | not applicable |
| Component library | in-house React components in `src/components/ui` and homepage sections in `src/components/homepage` |
| Icon library | `lucide-react`; keep existing icon allowlists for hero strip, CTAs, FAQ, and checklist icons |
| Font | `--font-display` Spectral for headings, `--font-sans` Hanken Grotesk for body/UI, `--font-mono` Space Mono for metadata/eyebrows |

Design-system source: `src/app/globals.css`, `src/components/ui/section/section.tsx`, `src/components/ui/button/button.tsx`, `docs/conventions.md`.

Phase 22 must preserve the existing homepage visual system. Prop-enable the current homepage components in place; do not introduce shadcn, new CSS modules, inline styles, cool grays, raw hex/rgb class values, or parallel CMS-only section components.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, card micro-gaps, focus offsets |
| sm | 8px | Compact element spacing, form field gaps |
| md | 16px | Default component gap, grid gap, card inner rhythm |
| lg | 24px | Section header/content separation, CTA groups |
| xl | 32px | Major intra-section spacing, article/card grids |
| 2xl | 48px | Compact section vertical padding on tablet/desktop |
| 3xl | 64px | Page-level section rhythm minimum |

Exceptions: keep existing approved homepage geometry and rhythm tokens exactly for parity: `px-gutter`, `py-section`, `max-w-base`, `max-w-wide`, `max-w-prose`, hero `min-h-[min(92vh,860px)]`, organic-herbs `min-h-110/md:min-h-120/lg:min-h-130`, card aspect ratios, 44px minimum icon/button touch targets, and pre-existing fractional Tailwind values already present in homepage components such as `gap-4.5`, `py-5.5`, `mb-6.5`, `px-6.5`, and `p-7.5`. Phase 22 may preserve these classes while prop-enabling content, but must not add new ad hoc spacing values.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | `type-body` — 16px | 400 | 1.6 |
| Label | `type-label` — 15.04px | 600 | 20px |
| Heading | `type-heading-01` — clamp(32px, 4vw, 54.4px) | 500 | 1.04 |
| Display | `type-display` — clamp(41.6px, 6.4vw, 86.4px) | 500 | 0.98 |

Typography contract: do not add new type utilities or heading scales in Phase 22. Route content must keep one visible `h1` in `HomepageHero`; section titles remain `h2`; card/article/FAQ titles remain `h3` or lower as currently implemented. Preserve existing `type-*` utilities and existing letter-spacing values; do not introduce additional fluid type, tracking, or font weights.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-paper` / `--color-paper: oklch(97.4% 0.008 92)` | Homepage wrapper, default section surfaces, body background |
| Secondary (30%) | `bg-paper-2`, `bg-card`, `bg-ink` | Sunken bands, cards, inverse newsletter/supply-chain bands, hero trust strip |
| Accent (10%) | `text-brand`, `bg-brand`, `text-gold`, `bg-gold-tint` | Primary CTA fills, CTA hover states, focus rings, hero/section eyebrows, checklist icons, FAQ open icon, link-arrow hovers |
| Destructive | `text-danger`, `bg-danger-tint`, `border-danger` | Form error messages only |

Accent reserved for: hero CTA, catalogue/contact/newsletter CTA states, section eyebrow highlights, link-arrow hover/focus states, proof-point icons, organic checklist icons, FAQ plus icon open state, form focus rings, and success/error feedback surfaces. Do not apply accent color to all body links or broad backgrounds.

Color contract: keep the warm/botanical palette. No cool gray additions. Authored Sanity images must sit inside the existing scrim/gradient treatments so text contrast remains at least as legible as the v1.5 homepage baseline.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Explore Our Teas` |
| Empty state heading | No public CMS empty state |
| Empty state body | Missing or invalid `homePage` content must fail route rendering; do not show static fallback sections or partial CMS placeholders. The existing Tea Journal empty article behavior may continue to omit that section only. |
| Error state | Public fallback copy, only if a route error boundary is used: `We could not load the homepage content. Please refresh or try again shortly.` Server logs/tests should identify the missing Sanity field. |
| Destructive confirmation | none — Phase 22 has no destructive user actions |

CTA and form labels to preserve:

| Surface | Contract |
|---------|----------|
| Hero | CTA label comes from CMS and seeded value remains `Explore Our Teas`; destination remains `/collections` unless CMS supplies another valid required link. |
| Product range cards | Existing hover action label remains `Shop now`; card title and destination come from CMS. |
| Private-label cards | Existing action labels remain CMS-driven seeded `Explore`. |
| Organic herbs | CTA remains CMS-driven seeded `Explore Our Herbs & Spices`. |
| Newsletter | Email placeholder `Enter your email`; submit label `Subscribe`; pending label `Subscribing…`; success `Thanks for signing up.`; error `Unable to send your signup right now. Please try again shortly.` |
| Contact | Field labels `Name`, `Number`, `Email`, `Message`; submit label `Submit`; pending label `Sending…`; success `Thanks. The Teavision team will review your enquiry shortly.`; default error `Unable to send your message right now. Please try again shortly.` |
| Tea Journal | CMS controls intro, blog handle, max posts, and link label; seeded link label remains `View all`; article cards remain live blog data. |
| Catalogue CTA | CMS must drive primary and secondary CTA copy; do not keep a hardcoded secondary action when CMS provides one. |

## Phase-Specific Rendering Contract

Section order is locked and must match the v1.5 homepage and Phase 21 schema:

1. `HomepageHero`
2. `ProductRange`
3. `HomepageNewsletter`
4. `PrivateLabel`
5. `OrganicHerbs`
6. `SupplyChain`
7. `CertificationCoverage`
8. `SupplyChainProtection`
9. `Testimonials`
10. `TeaJournal`
11. `ContactSection`
12. `Cta`
13. `Faq`

Rendering rules:

- Use Sanity content for every modeled Phase 21 field; do not leave modeled visible copy hardcoded in live route rendering.
- Keep forms/actions and decorative motif assets code-owned: newsletter/contact Server Actions, carousel behavior, teapot/label motifs, supply-chain handshake/stamp motifs, and catalogue cup/stamp motifs.
- Keep `Section.Root`, `Section.Container`, `Eyebrow`, `Button`, `OverlayImageCard`, `TestimonialsSlider`, and existing form primitives as the visual source of truth.
- Do not add a page-builder abstraction, section reordering, preview overlays, hidden static fallback, or duplicate CMS-only homepage component tree.
- Preserve existing Storybook story surfaces by moving static content into fixture/test/story data only, not runtime fallback.

## Image Contract

| Image Type | Contract |
|------------|----------|
| Hero LCP image | Sanity-authored image via `next/image` with `fill`, `sizes="100vw"`, `preload`, `fetchPriority="high"`, stable dimensions, crop/hotspot respected, and no deprecated `priority` prop. |
| Authored card/section images | Require usable asset URL plus width/height before render; preserve current aspect ratios, object-fit behavior, rounded corners, scrims, and `sizes` values. |
| Alt text | Use CMS alt text for meaningful authored images. Decorative code-owned motif images remain hidden or empty-alt as currently implemented. |
| Missing image data | Fail route render; do not guess dimensions, skip sections, or substitute static public images. |

## Interaction Contract

| Surface | Interaction |
|---------|-------------|
| CTAs and card links | Preserve current hover lift, arrow translate, focus-visible `ring-ring`, rounded-full button shape, and reduced-motion behavior. |
| Product range overlay cards | Preserve aspect-video mobile and aspect-square desktop geometry, bottom gradient scrim, hover CTA reveal, and always-visible CTA on touch/mobile. |
| Private-label cards | Preserve card lift/shadow on hover, numbered mono label, image block height, and link-arrow CTA. |
| Testimonials | Keep Embla carousel in the client leaf only; preserve arrow-key navigation, dot buttons, `aria-live` position announcement, and `role="region"` carousel labelling. |
| FAQ | Preserve native `details/summary`, plus icon rotation, keyboard focus ring, and no custom client state. |
| Newsletter and contact forms | Keep client interactivity in leaf form components only; preserve honeypot fields, pending disabled state, `aria-busy`, status/error announcements, and existing validation attributes. |
