# Phase 14 - UI Design Contract

> Visual and interaction contract for Shopify Customer Accounts. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none - use the existing local Tailwind 4 design system |
| Preset | not applicable |
| Component library | local primitives in `src/components/ui`; no shadcn registry |

Source: `AGENTS.md`, `src/app/globals.css`, `src/components/ui/*`, Phase 11 redesign contract, Phase 14 CONTEXT D-14.

### Component Contract

Use existing primitives before adding anything new:

| Need | Component / Pattern | Contract |
|------|---------------------|----------|
| Page bands | `Section.Root` | Account pages use `tone="surface"` or `tone="sunken"`; no marketing hero. |
| Page width | `Section.Container` | Use `variant="base"` for account workspaces, `variant="compact"` for bridge/explanatory pages. |
| Panels | `Card` | Use `tone="surface"` or `tone="sunken"`, `padding="lg"`, `radius="lg"` for dashboard, order, and address panels. |
| Actions | `Button` | Primary actions use `variant="brand"` for account/OAuth and save actions; secondary uses `variant="secondary"`; quiet text actions use `variant="quiet"`. |
| Icon-only header action | Link styled like the existing cart icon | Use `UserRound` or `CircleUserRound`; `aria-label="Account"`; href `/account`. Do not add a state-aware label in Phase 14. |
| Forms | `FormLabel`, `TextInput`, `Textarea`, `Select`, `Checkbox` | Use existing field focus, min-height, disabled, and placeholder styling. |
| Delete confirmation | Existing `Dialog` in a route-local client leaf | Required for address deletion only; include cancel and destructive submit actions. |
| Pagination | `Pagination` | Use for order history cursor pages; no infinite scroll. |
| Status pills | `Badge` or local pill markup | Use `organic`, `gold`, or neutral `bg-paper-2` treatment; preserve Shopify status accuracy. |

No CSS modules, inline styles, raw color classes, cool grays, or className concatenation. Use `cn()` for conditional class composition.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, field helper spacing, compact inline status gaps |
| sm | 8px | Control groups, table cell inner gaps, status pill gaps |
| md | 16px | Default card internals, form field stacks, mobile list spacing |
| lg | 24px | Card padding, dashboard grid gaps, form section breaks |
| xl | 32px | Desktop dashboard and account shell gaps |
| 2xl | 48px | Major account section breaks and top/bottom compact page padding |
| 3xl | 64px | Full page vertical breathing when no `py-section` band is appropriate |

Exceptions:

- Existing shared primitive internals remain as implemented: `IconButton`/header links use 44px hit targets (`size-11`), form controls use 48px minimum height (`min-h-12`), and `Button` keeps its established horizontal padding.
- Page gutters and vertical section rhythm use existing tokens `px-gutter` and `py-section`.
- Dense data rows may use 12px row gaps only inside order/address lists, but outer layout spacing must use the table above.

### Layout Rhythm

| Surface | Mobile | Desktop |
|---------|--------|---------|
| `/account` dashboard | Single column, `gap-6`, recent orders first | `grid-cols-[minmax(0,1fr)_22rem]`, `gap-8` or `gap-10`; main column recent orders, side column profile/default address/support |
| Address book | Stacked address cards, default first | 2-column card grid when width allows; default card spans first visual position |
| Address form pages | One-column form in compact container | Two-column field rows for name, city/state/postcode where useful; action row aligned right |
| Order history | Card list with summary rows | Semantic table with columns: order, date, payment, fulfillment, total, details |
| Order detail | Stacked sections | Summary card, line-item table/list, totals, shipping/tracking cards |
| Cart checkout context | Compact account-status block above checkout actions | Same block inside the order-summary/checkout area, not a dashboard panel |

## Typography

Account surfaces use four role sizes and two phase-level weights only: regular 400 and display medium 500. Existing primitive internals such as `Button` labels keep their built-in styling; do not add new font weights for Phase 14 account UI.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`type-body`) | 400 | 1.6 |
| Label | 14px (`type-body-sm` or existing `FormLabel`) | 400 | 1.6 |
| Heading | 19.2px (`type-heading-05`) | 500 | 1.1 |
| Display | 32-54.4px (`type-heading-01`) | 500 | 1.04 |

Usage:

- Page titles use `type-heading-01`, not `type-display`.
- Dashboard card titles use `type-heading-05`.
- Account summary copy, helper text, scoped errors, and empty-state body copy use `type-body` or `type-body-sm`.
- Order numbers, dates, status meta, form labels, and table headings may use existing `type-mono-meta`, but only for compact metadata.
- Do not use viewport-width font scaling outside the existing `type-heading-*` utilities.

## Color

Use token classes from `src/app/globals.css`; do not introduce raw hex/rgb/oklch classes in component markup.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-paper`, `text-ink` | Page background, account route shell, login bridge pages |
| Secondary (30%) | `bg-card`, `bg-paper-2`, `border-hairline`, `border-hairline-2` | Dashboard panels, address cards, order tables, loading skeletons, scoped section errors |
| Accent (10%) | `text-brand`, `bg-brand`, `bg-brand-tint`, `text-gold-deep`, `bg-gold-tint` | Primary account CTA, active/default indicators, selected pagination, focus/hover states, small status highlights |
| Destructive | `text-danger`, `bg-danger-tint`, `border-danger` | Address delete confirmation and address delete mutation errors only |

Accent reserved for:

- `Sign in with Shopify`, `Save address`, `Update profile`, `Retry checkout`, and other primary recovery/save actions.
- Default address marker and successful mutation confirmation.
- Active order pagination item.
- Header account icon hover/focus treatment.
- Compact cart account status and blocked-checkout recovery action.
- Eyebrow/meta accents where a section needs hierarchy.

Do not use accent color for every link or every table row. Ordinary account navigation, secondary links, and support links remain `text-ink`, `text-ink-soft`, or `text-brand` only on hover.

| State | Copy Contract |
|-------|---------------|

## Interaction Contract

### Header And Footer Account Entry

- Add one icon-only account entry to the existing header right-side icon cluster beside search and cart.
- The account icon always links to `/account`, including mobile. Do not add a duplicate mobile-menu account row by default.
- Use a lucide account icon at `size-4`, `strokeWidth={1.8}`, 44px hit target, and the same hover/focus treatment as the search/cart icons.
- Replace the footer `Login` href from `https://mrtea.com.au/account/login` to `/account`; keep the visible label `Login` unless a later owner decision changes footer copy.

### Login, Bridge, And Auth States

- `/account/login` is a functional OAuth start page, not a classic password form and not a marketing landing page.
- The page uses a compact card/panel with a short explanation and the primary CTA `Sign in with Shopify`.
- Legacy register/reset/password routes bridge to modern OAuth with explanatory copy and support fallback. Do not show username/password inputs.
- OAuth callback verification failures render a safe error panel with `Start sign-in again`; do not expose provider details or token values.

### Account Dashboard

- `/account` leads with recent orders, then default address and profile/account summary.
- Use a practical workspace layout: panels, tables, lists, and compact support content. No oversized hero, decorative gradients, or promotional wholesale upsell.
- Logout is present as a secondary account action, visually quieter than order/address/profile actions.
- If one account section fails, keep the other sections visible and show a section-level `role="alert"` message inside the failed panel.

### Orders

- Order history uses predictable pagination with `Pagination`; no infinite scroll.
- Desktop order history uses a semantic table. Mobile collapses into cards with the same fields.
- Required order summary fields: order name/number, date, financial status, fulfillment status, total, and details link.
- Order detail pages show line items, quantities, prices/totals, financial and fulfillment status, shipping address when available, and tracking links only when Shopify provides them.
- Status labels must be customer-friendly but faithful to Shopify data. Do not invent success states.

### Addresses And Profile

- Address book sorts the default address first.
- Address add/edit use dedicated pages: `/account/addresses/new` and `/account/addresses/[id]/edit`.
- Forms are Australia-first by field order and defaults, while still allowing Shopify-supported international addresses.
- Field errors render next to fields when Shopify provides a field path; otherwise show a form-level `role="alert"` message.
- Address deletion requires a confirmation dialog before submitting the destructive mutation.
- If default-address mutation succeeds but refresh fails, show a success confirmation plus scoped refresh guidance rather than treating the mutation as failed.

### Cart Account Context

- Cart stays a cart. Add only a compact account-status line near checkout controls.
- Signed-in copy uses restrained language such as `Checking out with your Teavision account`.
- If pre-checkout buyer identity sync fails, block checkout. The primary recovery is `Retry checkout`; secondary actions are `Sign in again` and `Contact support`.
- Do not offer a silent guest-checkout fallback after buyer identity sync failure.

### Loading, Empty, And Error States

- Route loading states use stable skeleton blocks with `bg-paper-2`, `rounded-lg`, and fixed heights that do not shift the account shell.
- Full-page account errors use paper background, `type-heading-02` or `type-heading-03`, muted copy, and `Button` actions.
- Form pending states keep the button label area stable and set `aria-busy` via `Button isLoading`.
- Use `role="status"` or `aria-live="polite"` for mutation success messages; use `role="alert"` for validation and blocking errors.

## Accessibility Contract

- Every icon-only control must have an accessible name.
- Minimum interactive target is 44px for header icon links and icon buttons; form controls stay at 48px minimum height.
- Focus uses existing `focus-visible:ring-ring`, `focus-visible:ring-2`, and `focus-visible:ring-offset-2` or existing field `shadow-focus`.
- Tables keep semantic headers on desktop. Mobile card views must preserve labels for each value.
- Dialog delete confirmation traps focus, closes on Escape, restores focus, and uses `aria-modal`.
- Address forms use `autoComplete` values where applicable: `given-name`, `family-name`, `address-line1`, `address-line2`, `address-level2`, `address-level1`, `postal-code`, `country`, `tel`.
- Error text is tied to controls with `aria-describedby`.
- Reduced motion is already globally enforced; do not add account-specific essential motion.
