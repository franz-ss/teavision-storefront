<objective>
Add profile editing and address self-service: supported profile/contact mutations, default-first address book, add/edit/delete/default address pages, accessible address forms, delete confirmation, and Shopify user-error normalization.
</objective>

<threat_model>
- T-14-05 Account mutation authorization, severity high: every profile/address Server Action calls `requireCustomerAccountSession()` and never trusts client-provided customer IDs.
- T-14-02 Token exposure, severity high: mutation state returned to client components contains only normalized field/form messages, never tokens, raw Shopify response bodies, or PII beyond the submitted form display.
- T-14-07 Customer data caching leak, severity high: address/profile refreshes are request-scoped and use `revalidatePath('/account')` or account subpaths, not cache tags shared with product data.
</threat_model>

  <action>
Extend Customer Account operations with:
- `updateCustomerProfile(session, input)` for first name, last name, and phone/email fields only when supported by Customer Account API.
- `createCustomerAddress(session, input)`.
- `updateCustomerAddress(session, addressId, input)`.
- `deleteCustomerAddress(session, addressId)`.
- `setDefaultCustomerAddress(session, addressId)` implemented through address update with `defaultAddress: true`.
- `normalizeCustomerAccountUserErrors(errors)` returning `{ fieldErrors: Record<string, string>; formError: string | null }`.

Create `actions.ts` with `'use server'` and exports:
- `updateProfileAction(previousState, formData)`
- `createAddressAction(previousState, formData)`
- `updateAddressAction(previousState, formData)`
- `deleteAddressAction(previousState, formData)`
- `setDefaultAddressAction(previousState, formData)`

Every action must call `requireCustomerAccountSession()` before parsing mutation IDs and must call `revalidatePath('/account')` plus the relevant subpath after success.
  </action>


  </verify>
  <acceptance_criteria>
    - `actions.ts` starts with `'use server'`.
    - Each exported action contains `requireCustomerAccountSession`.
    - `normalizeCustomerAccountUserErrors` maps Shopify field `['address1']` to field error key `address1`.
    - Tests assert missing session returns an unauthorized form state and does not call Customer Account mutations.
    - No action accepts or reads a `customerId` from `FormData`.
  </acceptance_criteria>
  <done>Profile and address mutations are authorized, normalized, and safe to invoke through forms.</done>
