<threat_model>
Trust boundaries:

- Inputs from live/current crawls, Shopify/Sanity route helpers, sitemaps, and optional owner/SEO exports cross into code only after confirmation.
- Local app redirect proof is separate from DNS, Vercel, production host, alternate-host, Shopify-domain, and Search Console proof.

STRIDE register:

- T-18-01 spoofing/tampering: an inferred redirect could send traffic to the wrong destination. Mitigation: require two-source confirmation and record evidence source/date before adding app redirects.
- T-18-02 denial-of-service: broad redirect fallbacks or loops can harm crawlability. Mitigation: add only explicit path redirects and probe each app-owned redirect with `redirect: manual`.
- T-18-03 information disclosure: owner exports or crawl data may contain private Search Console details. Mitigation: commit only sanitized route/path decisions and no query, credential, customer, or token data.
  </threat_model>
