## Task 3: Add a light default-listing query/helper

type: implementation

files:

- `src/lib/sanity/queries/blog.ts`
- `src/lib/sanity/types.ts`
- `src/lib/blog/operations.ts`

action:

- Add a separate GROQ query for the unfiltered default listing.
- Fetch only:
  - blog metadata, SEO, hero image
  - configured featured posts
  - first page of latest published article summaries
  - enough count metadata to preserve pagination UI
- Avoid fetching `bodyText` for article summaries if excerpt data is sufficient; if reading time needs body text, keep body text only where required and document the tradeoff.
- Return a shape separate from `SanityBlogListingResult` so the full-blog fallback stays untouched.
- Add a new cached operation, for example `getDefaultBlogListing()`, tagged with `blog` and `blog-${normalizedHandle}`, with `cacheLife('hours')`.

verify:

- TypeScript build.
- Compare default listing visible articles against the existing path for `/blogs/teavision-blogs` where feasible.

acceptance_criteria:

- Default listing no longer needs to fetch every article summary before rendering the first page.
- Existing `getBlog()` remains available for tag/search fallback.
