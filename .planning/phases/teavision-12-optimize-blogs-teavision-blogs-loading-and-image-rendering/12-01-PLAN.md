type: implementation

files:

- `src/lib/sanity/types.ts`

action:

- Add `lqip: string | null` to `BlogImage`.
- Confirm `SanityImageAsset.metadata.lqip` remains typed as `string | null`; do not introduce `any`.

- Keep the existing fallback rule: return `null` if there is no usable asset URL or generated Sanity URL.

verify:

- Search references to `BlogImage` and update any required type expectations.

## Task 2: Add bounded Sanity image URL options

type: implementation

files:

- `src/lib/sanity/client.ts`
- `src/lib/blog/operations.ts`

action:

- Extend `getSanityImageUrl()` with a typed options object for `width`, `height`, `quality`, and `fit`.
- Use integer dimensions only.
- Default bounded Sanity URLs to `auto('format')` and `fit('max')` unless a caller explicitly needs another fit.
- Introduce use-case-specific image shaping in `reshapeImage()` or a small colocated helper:
  - hero images get a full-width maximum suitable for the listing hero.
  - featured article cards get a medium maximum.
  - default article cards get a smaller maximum.
- Keep quality values aligned with `next.config.ts` (`68` or `75`).

verify:

- Inspect generated Sanity image URLs in browser devtools or route HTML for `auto=format`, bounded `w`, and expected `q`.
- `pnpm lint`.

acceptance_criteria:

- Sanity URLs are no longer unbounded original asset URLs for blog listing hero/card images.
- The implementation does not crop editorial content unexpectedly or upscale small assets.
