## Phase Interaction and Semantic Contract

### Product specification disclosures

- Retain each existing native `<details>` and its native `<summary>` as the sole interactive disclosure control.
- Render the visible disclosure title in an `<h2>` inside that summary. The H2 provides document-outline meaning; it is not a replacement control and must not receive a click handler, `role="button"`, or `aria-expanded`.
- Preserve the first disclosure as open by default, all existing labels and copy, the decorative `ChevronDown` (`aria-hidden="true"`), its rotation, and the current keyboard/native open-close behaviour.
- Preserve the current summary layout: flex row, title wrapping/shrinking safely beside the non-shrinking chevron, cursor, and visible focus ring. Do not visually restyle the label when moving its typography class to the H2.
- The product page continues to expose exactly one page H1. This change adds only the specification-title H2s.

### Collection story/read-more disclosures

- Retain `StoryDisclosure` as native `<details>` / `<summary>` with its existing default-closed behaviour, focus ring, open-state chevron motion, padding, border, and mobile overflow protection.
- The `RichText` `disclosure` value remains a presentation variant only; it must not become a sanitizer variant or alter interaction behaviour.
- Only collection story HTML uses the new `collectionStory` sanitizer. Product descriptions continue to use the unchanged `compact` sanitizer entry point and output contract.

| Source heading in collection story HTML | Required output element | Visual treatment |
|-----------------------------------------|-------------------------|------------------|
| H1 / H2 | H3 (existing compact-safe treatment) | Preserve current compact H3 appearance |
| H3 | H2 | Preserve current `type-heading-05 text-ink mt-5` appearance |
| H4 | H3 | Preserve current `type-label text-ink mt-5` appearance |
| H5 / H6 | unchanged | Preserve existing safe behaviour and styles |

The sanitizer must retain the existing shared allowlist, safe link/image handling, class filtering, and table scroll wrappers. No new `dangerouslySetInnerHTML` boundary or arbitrary `SanitizedHtml` cast is permitted in application code.

## Typography

This phase preserves the existing typography hierarchy while correcting the semantic element names. The phase-specific contract declares two weights only: medium (500) and semibold (600).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Product disclosure H2 | 1.15rem (18.4px) | 500 | `leading-tight` (1.25) |
| Collection story H2 (source H3) | 1.2rem (19.2px) | 500 | 1.1 |
| Collection story H3 (source H4) | 0.94rem (15px) | 600 | 1.25rem |

All other body, H1, and imported H5/H6 typography remains unchanged. Never use the semantic promotion to increase font size, introduce a new font, or alter the product compact-description typography.
