<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser events to analytics destinations | Event payloads leave the application boundary only after consent and env gates pass |
| Local/CI tests to production destinations | Tests must use fake sink and never send production analytics events |
| Lead Server Actions to analytics event layer | Submitted PII is available server-side and must not be copied into analytics payloads |
| CSP policy to external script hosts | Host allowlists must not be widened unless destinations are explicitly configured |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-07 | Information Disclosure | analytics destination loader | mitigate | Gate external scripts by consent and public env configuration |
| T-16-08 | Tampering | CI analytics data | mitigate | Fake/test sink for local and CI; tests assert real destinations are absent unless env-enabled |
| T-16-09 | Information Disclosure | lead event payloads | mitigate | Closed event union excludes PII fields and emits only lead kind/status metadata |
| T-16-10 | Elevation of Privilege / Tampering | CSP analytics hosts | mitigate | Env-gated host additions with tests for empty and configured states |

</threat_model>
