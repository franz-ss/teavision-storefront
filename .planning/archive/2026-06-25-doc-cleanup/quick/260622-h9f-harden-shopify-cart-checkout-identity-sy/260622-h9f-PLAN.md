# Quick Plan: Harden Shopify cart checkout identity sync and retained-cart behavior

## Objective

Make the current Shopify cart flow safer without introducing account-cart merge semantics:

- Checkout handoff must use the cart returned after buyer identity sync when available.
- Login callback must surface cart identity sync failure when the shopper returns to cart.
- Logout fallback must make retained browser cart behavior explicit.

## Scope

1. Add tests that capture the expected checkout URL, callback redirect, and logout notice behavior.
2. Update the cart Server Action to retain the post-sync Shopify cart response.
3. Update account callback/logout/login copy paths for visible failure and retained-cart signals.
4. Run targeted cart/account tests plus the relevant integration suite.

## Out of Scope

- Account cart persistence tables or a Teavision-side cart database.
- Guest/authenticated cart merging beyond Shopify's existing cart buyer identity behavior.
- Hosted checkout, payment, shipping-rate, tax, order-creation, or success-redirect testing.
