## Phases

### Phase 20: PageSpeed 100/100 Perfection (lean scope)

**Goal (lean, per D-16):** Resolve the four homepage `/` PSI screenshot findings — (1) hero LCP `fetchPriority="high"`, (2) AVIF image delivery, (3) lazy-loaded client Sentry SDK, (4) a measured `experimental.inlineCss` decision — minimal, architecture-preserving, no regressions.
**Requirements (in scope):** PSI-03, PSI-05, PSI-06, PSI-07, PSI-08
**Requirements (deferred per D-16):** PSI-01, PSI-02, PSI-04, PSI-09, PSI-10, PSI-11, PSI-12, PSI-13, PSI-14, PSI-15
**Plans:** 1

Plans:

- [x] 20-01 — Homepage `/` PSI screenshot fixes (hero fetchPriority + AVIF + lazy Sentry SDK + measured inlineCss decision + regression sweep + evidence)

**Details:**

- **Hero LCP** — `src/components/homepage/hero/hero.tsx` gained `fetchPriority="high"` (kept `preload`, `fill`, `sizes="100vw"`, `alt=""`) so the emitted `<link rel="preload" as="image">` and the rendered `<img>` both carry `fetchpriority=high`. Verified against Next.js 16 runtime source (`get-img-props.js`, `image-component.js`), not just docs prose. `fetchPriority="high"` is the permitted LCP prop, distinct from the banned deprecated `priority` (D-15 / D-10 refined).
- **AVIF** — `next.config.ts` `images.formats: ['image/avif','image/webp']`; AVIF-capable browsers receive the existing AVIF hero directly (confirmed `Content-Type: image/avif` served in production).
- **Lazy Sentry** — `instrumentation-client.ts` rewritten to dynamically `import('@sentry/nextjs')` only inside the `NEXT_PUBLIC_SENTRY_DSN` guard, removing the ~142KB Sentry browser SDK from every cold route load; `onRouterTransitionStart` stays exported as a typed delegating no-op (no `any`); a `.catch()` handles the dynamic import (WR-01 fix `1ba2ef8e`).
- **inlineCss (measured skip)** — `experimental.inlineCss` was later enabled, locally measured (LCP/FCP/SI all regressed), and reverted; documented as a measured tradeoff in `docs/launch/homepage-performance-fixes.md`. The 540ms render-blocking CSS finding is an accepted, documented tradeoff.
- **Follow-ups** — below-the-fold Tea Journal preload removed (`e0ba5ec0`); homepage a11y colour-contrast fixes; CSP console-error fix (dropped inert `upgrade-insecure-requests` from report-only CSP).
- **Contract test** — `scripts/component-contracts/launch-image-performance.test.mjs` exempts only the homepage hero from the preload+fetchPriority guard; product-gallery, product-card, and both collection heroes keep the guard.

**Real owner-run PSI (2026-07-01, `docs/launch/homepage-performance-fixes.md` + `20-HUMAN-UAT.md`):** Performance 95–97, LCP 3.4s→3.0s (residual gap diagnosed as a Lantern simulate-throttling lab artifact — hero paints ~150ms), Speed Index 5.1s→1.9s, TBT 90ms→30ms, CLS 0 held, Accessibility 100, Best Practices 100.

**Verification:** `20-VERIFICATION.md` status `verified`; 5/5 lean must-haves verified at code level; the one human-gated item (real post-deploy PSI on `/`) completed and resolved 2026-07-01 (owner-run). No regression of Phase 18 crawlable-HTML or Phase 19 single-visible-H1 guards.
