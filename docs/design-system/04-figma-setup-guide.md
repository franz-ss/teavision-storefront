# Figma Setup Guide — Teavision Design SystemUse this as your roadmap as you build the Figma file. Each step is small, ordered, and matches the spec in `02-foundations.md` and `03-components.md`. After each step you can share the file URL and I'll inspect what you built.---## Step 0 — One-time setup (5 minutes)### Install Figma DesktopThe MCP plugin I use to inspect your file requires Figma Desktop, not just the web app.- Download from <https://www.figma.com/downloads/>- Install and sign in with the same account you use for the web### Enable Dev Mode MCP serverIn Figma Desktop, with any file open:1. Click the Figma logo (top-left) → **Preferences**2. Find **Enable local MCP server** (sometimes labeled **Dev Mode MCP server**) and toggle it on3. You'll see a confirmation that the local server is running on `localhost:3845`That's it for setup. If you don't see this option, your Figma Desktop may need updating to the latest version (early-2025 or newer).---## Step 1 — Create the file (2 minutes)1. In Figma → **+ New design file**2. Suggested frames (left to right, top to bottom):1. **Color — Primitives** — for each scale (steep, stone, ink, amber, status), a row of 11 swatches with name + hex2. **Color — Semantics (Light)** — semantic tokens grouped by surface/text/border/action with the actual color filled3. **Color — Semantics (Dark)** — same, but in dark mode4. **Typography** — every text style demonstrated with a sample line ("Australia's #1 tea company")5. **Spacing** — bars showing each space token's pixel value6. **Radius** — a row of squares showing each radius7. **Elevation** — a row of cards each showing one shadow level8. **Motion** — a static reference (since motion is hard to represent statically) listing duration + easing tokensThis page becomes the **single source of truth** for the design system. Anyone opening the file gets it immediately.---## Step 7 — Build components (priority order)Suggested order — work through `03-components.md` from top to bottom but skip ahead to the first 12 priority components:1. Button (5 variants × 3 sizes × 5 states = the master atomic component)2. Input + Form Field3. Badge + Tag + Certification Badge4. Card (base) → Product Card → Article Card5. Header (with Mega Menu)6. Footer7. Stat Block8. Testimonial Card9. FAQ Accordion10. Quote Request Form (pattern, composed)11. Spec Table12. Modal + DrawerAnything past that is sub-page or v1.1 territory.### Component-build conventionsFor every component:- Build the **default** state first.- Use **Component Properties** (Variant, Boolean, Instance Swap, Text) — not separate components per state.- Wire **all** color/typography/spacing properties to the variables/styles you set up. Never paste a hex.- Name layers semantically: `button/icon-leading`, `card/media`, `field/helper-text`. Future you (and your devs) will thank you.- Add a **description** to each component master with the props list and link back to the relevant section in `03-components.md`.---## Step 8 — Where I come inAfter you've finished any of these steps and want a review, paste the Figma URL or a specific frame URL into the chat (right-click any frame → **Copy link to selection**, gives a URL with `?node-id=…`). I'll inspect via the MCP plugin and give targeted feedback:- **Wrong token usage** — "this fill is `steep/600` directly; it should be `bg/brand` semantic"- **Drift from the spec** — "`Button md` should be 40 tall, you have 36"- **Missing variants** — "your Card has no Interactive variant; it's spec'd"- **Naming inconsistencies** — "`btn-primary` here vs `button/primary` elsewhere; pick one"Aim for: build a chunk → review → adjust → build the next chunk.
