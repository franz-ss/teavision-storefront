For current runtime tokens and component guidance, use `app/globals.css` and `DESIGN.md`. This file preserves the earlier full token package and may mention dark themes, warning/info statuses, legacy aliases, hex values, and token scopes that are no longer part of the minimal runtime token surface.

# FoundationsThe token system that everything else inherits from. Two layers:- **Primitives** (`--color-steep-700`, `--space-4`): raw values. Designers should rarely touch these directly.- **Semantics** (`--color-bg-brand`, `--color-text-default`): usage-based aliases. Components consume only semantic tokens.This separation lets us re-point semantics (e.g. swap brand color) without touching every component.---## 1. Color### Color philosophy in one paragraphWe use color in three layers: **brand surfaces** (deep "Steep" green for hero areas, the mark, brand moments), **action** (near-black ink for primary buttons and high-contrast actions), and **accent** (warm amber, reserved for certifications, sale tags, and small moments of energy). Status colors (success, warning, danger, info) are slightly desaturated to read functional rather than decorative. Light theme is the default; dark theme is a system-respecting option.### Primitive scalesEach scale has 11 stops (50 → 950).
