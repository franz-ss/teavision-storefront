# SEO URL Parity Register

This register maps the Phase 18 SEO audit URL-parity finding to local app
redirects, no-op parity paths, and owner/operator handoff work. It intentionally
commits path decisions only, not private analytics exports, protected crawl
exports, checkout destinations, customer or order details, secrets, or
credentials.

## Two-source confirmation rule

An app-owned redirect can be encoded or retained in code only when both evidence
columns are populated with stable, non-private sources and the `Decision` column
is exactly `app-owned redirect`. One source must confirm the legacy/current
source shape, and the other must confirm the canonical target route or existing
route registry. Missing owner exports, inferred slug guesses, host-level
redirects, DNS/Vercel redirects, alternate-host redirects, Shopify-domain
redirects, and Search Console-only evidence stay in `## Owner/operator handoff`.
